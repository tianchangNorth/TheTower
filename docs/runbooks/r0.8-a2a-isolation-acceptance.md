# R0.8 A2A isolation 验收手册

> 状态：验收工具已实现；Mock/fixture 自动化通过。Codex、Claude 真实报告需要操作者明确确认将验收 prompt、平台 system prompt、启用的 Skills 和最小 Thread 上下文发送给对应外部模型服务后运行。

## 1. 验收目标

每个真实 Provider 独立验证以下事实：

1. Runner 的 final/stdout 只写入 `agent_stream`，不会生成隐式公开 callback；
2. Agent 必须通过 `mcp__thetower__post_message` 显式写入 public/private callback；
3. private callback 只进入发送者与指定接收者上下文；
4. `play` 对所有 Agent 隐藏 `agent_stream`；
5. `debug` 可共享普通 stdout/tool output，但会从其他 Agent 上下文剥离 thinking；
6. stream 按 `(thread, invocation, agent)` 合并为一行，并接受 payload 阈值检查。

## 2. 真实 Codex/Claude 验收

前置条件：

- 已安装并登录 `codex` / `claude`；
- 已理解真实验收会调用外部模型并消耗相应订阅或 API 配额；
- 已审阅上述数据披露范围；
- 全仓已完成构建，MCP Server 可从 `packages/mcp-server/dist/index.js` 启动。

```bash
pnpm build
pnpm test:e2e:real
```

只验收单个 Provider：

```bash
THE_TOWER_REAL_E2E_PROVIDERS=codex pnpm test:e2e:real
THE_TOWER_REAL_E2E_PROVIDERS=claude pnpm test:e2e:real
```

如 Provider CLI 的默认模型不可用或与本机 CLI 版本不兼容，可显式指定验收模型：

```bash
THE_TOWER_REAL_E2E_PROVIDERS=codex THE_TOWER_REAL_CODEX_MODEL=<兼容模型> pnpm test:e2e:real
THE_TOWER_REAL_E2E_PROVIDERS=claude THE_TOWER_REAL_CLAUDE_MODEL=<兼容模型> pnpm test:e2e:real
```

显式模型会通过 Runner 的 `--model` 参数传给 Provider CLI；未设置时沿用对应 CLI 的默认模型。报告会记录最终采用的是显式模型还是 CLI 默认值。

可用变量：

| 变量 | 默认值 | 用途 |
| --- | --- | --- |
| `THE_TOWER_REAL_E2E_PROVIDERS` | `codex,claude` | 选择真实 Provider |
| `THE_TOWER_REAL_E2E_TIMEOUT_MS` | `300000` | 单 Provider 超时 |
| `THE_TOWER_ACCEPTANCE_REPORT_DIR` | `test-results/real-a2a` | JSON 证据目录 |
| `THE_TOWER_REAL_CODEX_MODEL` | Codex CLI 默认值 | 覆盖 Codex 验收模型 |
| `THE_TOWER_REAL_CLAUDE_MODEL` | Claude CLI 默认值 | 覆盖 Claude 验收模型 |

测试使用临时 Workspace、内存数据库和随机 loopback 端口。每个 Provider 生成独立 JSON，包含 callback、play/debug 上下文和 stream storage 证据；临时 Workspace 在结束后删除。

真实验收不进入默认 `test:ci`，避免 CI 无凭证时误调用外部模型。发布前应由受信任操作者运行并归档报告。

## 3. 历史数据库 migration 演练

先构建 API，然后对生产数据库的 SQLite 在线 backup 副本执行 migration：

```bash
pnpm build
pnpm rehearse:migration -- --db /path/to/app.db --output /path/to/rehearsal-output
```

演练不会修改源数据库。输出目录包含：

- 带时间戳的迁移后 SQLite 副本；
- `a2a-migration-rehearsal-report.json`；
- `agent_final → callback`、`debug → play`、migration version 和二次执行幂等性检查。

退出码 `0` 表示全部验证通过，`2` 表示演练完成但存在验证失败。

## 4. Stream 落库量观察

```bash
pnpm build
pnpm observe:streams -- --db /path/to/app.db --output /path/to/stream-storage-report.json
```

默认预算：

- 每个 `(thread, invocation, agent)` 最多 `1` 行 `agent_stream`；
- 单组序列化 payload 最大 `1 MiB`。

可覆盖阈值：

```bash
pnpm observe:streams -- --db /path/to/app.db --max-rows 1 --max-bytes 2097152
```

报告给出 SQLite 分配空间、stream 总行数、分组数、总 payload、P95/最大 payload，以及具体超限组。发现超限时退出码为 `2`，便于纳入定期运维检查。
