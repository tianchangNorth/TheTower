# R0.9 文档真相源校正验收记录（2026-07-22）

## 结论

**正式通过，R0.9 已关闭。**

文档已形成明确的事实优先级：能力矩阵回答“是否支持”，Current 架构回答“如何工作”，Roadmap 回答“下一步做什么”，ADR 记录“为什么这样决定”，Runbook/Acceptance 保存可复现步骤与验收证据。历史计划不再承担当前状态说明。

## 本轮校正

已修正以下高风险冲突：

| 冲突 | 旧描述 | 当前事实 |
| --- | --- | --- |
| 未实现 Provider | Gemini/OpenAI API/Custom 回退 Mock | 返回 `unsupported_agent_provider`，不静默降级 |
| Route mode | `fanout`/`parallel` 接受后顺序执行 | 新请求返回 `unsupported_route_mode`；当前只支持 `single`/`serial` |
| Codex 默认权限 | `danger-full-access + on-request` | `read-only + untrusted`，callback network 默认关闭 |
| Claude 默认权限 | `bypassPermissions` | `default` |
| API 控制面 | 宽松 CORS、无远程门禁 | 默认本地 origin；非 loopback 必须配置 Operator Token |
| Callback 身份 | invocation token + 请求体 Agent 身份 | grant 绑定 invocation 与 Agent，服务端从认证结果构造 caller |
| SSE replay | 未实现、只靠重连刷新 | SQLite event log + `id:` + `Last-Event-ID` replay + sync/heartbeat |
| A2A 输出语义 | final/callback/stream 过渡规则混杂 | stdout/final 统一为 `agent_stream`；正式发言只走显式 `callback` |
