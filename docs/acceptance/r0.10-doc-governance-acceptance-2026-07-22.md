# R0.10 文档 metadata/lint 验收记录（2026-07-22）

## 结论

**正式通过，R0.10 已关闭；R0 契约统一与发布基线全部完成。**

项目现在使用 `docs/metadata.json` 管理文档状态和关系，`docs/README.md` 由脚本生成。CI 会拒绝 metadata 漂移、遗漏文档、重复真相源、无替代入口的 Superseded 文档、断链和过期生成索引。

## Metadata schema v1

每份非生成 Markdown 文档必须声明：

- `path`：仓库内 `docs/` 规范路径；
- `title`：必须与文档首个 H1 完全一致；
- `status`：`current | accepted | completed-record | reference | superseded`；
- `kind`：roadmap、architecture、ADR、acceptance、phase 等受控类型。

附加规则：

- `current` 必须声明 `lastVerified` 和至少一个唯一 `truthScopes`；
- `superseded` 必须声明 `replacedBy`，替代文档必须存在、受管且不能同为 Superseded；
- `superseded` 文件正文必须保留可见的 `文档状态：Superseded` 横幅；
- `accepted` 只允许用于 ADR；
- `docs/README.md` 是生成文件，不进入 metadata 文档列表。

## 命令与 CI

```bash
pnpm docs:lint      # metadata、覆盖率、替代链、truth scope、相对链接
pnpm docs:generate  # 从 metadata 生成 docs/README.md
pnpm docs:check     # lint + 生成结果防漂移检查
```

`pnpm test:ci` 的第一步为 `pnpm docs:check`。因此新增或重命名 Markdown、修改标题/状态、删除替代来源，或忘记重新生成索引都会阻断发布门禁。

- Current truth scope 无重复 owner；

- 全仓 Markdown 相对链接通过；
- 自动生成索引与 metadata 完全同步；
- `docs:check` 已接入根 `test:ci` 和 GitHub Actions 既有发布入口；
- 脚本通过 Node 语法检查，仓库通过 whitespace 检查。

完整执行 `pnpm test:ci` 通过：docs check、全仓 lint、production build、170 个单元测试、3 个 API integration、2 个 migration 和 6 个 production Playwright 主链全部成功。

## 维护流程

1. 新增文档时先在 `docs/metadata.json` 注册；
2. 修改 H1、状态或替代关系时同步修改 metadata；
3. 运行 `pnpm docs:generate`；
4. 提交前运行 `pnpm docs:check`；
5. Current 文档的事实发生变化时更新 `lastVerified`。
